## 컨텍스트

서버 데이터 패칭, 캐싱, 동기화를 수동으로 관리하면 보일러플레이트 코드가 많아지고

## 결정

서버 상태 관리 라이브러리로 React Query(TanStack Query)를 사용한다.

## 이유

- 캐싱, 백그라운드 재검증, 낙관적 업데이트를 선언적으로 처리
- Redux 대비 서버 상태에 특화되어 보일러플레이트 최소화

**허용**
- `useQuery`, `useMutation`으로 서버 데이터 패칭
- QueryClient 인스턴스를 통한 캐시 직접 조작

**금지**
- 서버 상태를 Redux/Zustand에 저장 (클라이언트 상태와 혼합 금지)
